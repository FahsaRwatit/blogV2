# Go GC 机制深度解析

## 三色标记算法

GC 将所有对象分为三种颜色：

- **白色**：未被标记，可能是垃圾
- **灰色**：已被标记，但引用的对象还未扫描
- **黑色**：已被标记，且所有引用的对象都已扫描

```go
// 手动触发 GC
runtime.GC()

// 设置 GC 百分比（默认 100，内存翻倍时触发）
debug.SetGCPercent(200)
```

## 写屏障

Go 1.8 引入了**混合写屏障**，解决了并发 GC 中对象引用被修改的问题。

## 调优建议

1. 合理设置 `GOGC` 环境变量
2. 使用 `sync.Pool` 复用临时对象
3. 用 `pprof` 定位内存分配热点

```bash
GOGC=200 ./myapp
```

> GC 调优核心：**减少分配，延迟回收，复用内存**。
